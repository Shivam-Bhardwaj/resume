# Job Description Index for Resume Tailoring

This index groups canonical job descriptions into categories aligned with your skills and experience. Use the categories to decide which projects, bullets, and skills to foreground when generating a targeted resume.

## 1. Autonomy, SLAM & Perception Robotics Software

Focus: ROS/ROS2, SLAM, sensor fusion, perception, embedded deployment.

Representative JDs:

- Senior Robotics Software Engineer – Autonomy & Perception (`senior-robotics-software-engineer-autonomy.md`)
- SLAM Engineer (example) (`slam_engineer.md`)

When to use:
- Roles titled "Robotics Software Engineer", "Autonomy Engineer", "Perception Engineer", "SLAM Engineer".
- Teams building navigation, mapping, or perception stacks for mobile robots, drones, or AVs.

Emphasize:
- ROS/ROS2 (Nav2, MoveIt), C++14/17, Python
- SLAM (GMapping, Cartographer), sensor fusion, Kalman filters
- Lidar / camera integration, embedded edge deployment (Jetson, x86)

## 2. Robotics Systems, Sensors & Integration

Focus: cross-disciplinary systems work, sensor stacks, field integration, and validation.

Representative JDs:

- Robotics Systems Engineer – Sensors & Integration (`robotics-systems-engineer-integration.md`)
- Robotics Systems Engineer, Optimus – Tesla (example) (`tesla-robotics-systems-engineer-optimus.md`)

When to use:
- Roles titled "Robotics Systems Engineer", "Mechatronics Engineer", "Field Robotics Engineer".
- Positions emphasizing system bring-up, cross-team debugging, and on-site deployments.

Emphasize:
- Multi-sensor integration (Lidar, cameras, IMUs, encoders)
- Industrial protocols (EtherCAT, Modbus, CAN), PLC / Beckhoff TwinCAT
- Test rigs, fixtures, validation, and cross-functional coordination

## 3. Embedded Firmware & Motion Control

Focus: MCU firmware, real-time control, low-level drivers, safety.

Representative JDs:

- Embedded Firmware Engineer – Motion Control (`embedded-firmware-engineer-motion-control.md`)

When to use:
- Roles titled "Embedded Firmware Engineer", "Controls Engineer", "Motion Control Engineer".
- Jobs emphasizing MCU work, motor control, and low-level communication.

Emphasize:
- Embedded C, Rust (embedded-hal), STM32/ESP32/Ambiq
- Motor control (servo/BLDC/stepper), PID, real-time constraints
- RTOS or bare-metal development, debugging with scopes and logic analyzers

## 4. Mechanical Design & Robotics Hardware

Focus: CAD-heavy roles for robotics, fixtures, and AR/VR hardware.

Representative JDs:

- Senior Mechanical Design Engineer – Robotics Hardware (`senior-mechanical-design-engineer-robotics-hardware.md`)

When to use:
- Roles titled "Mechanical Design Engineer", "Hardware Design Engineer", "AR/VR Mechanical Engineer".
- Jobs emphasizing CAD, GD&T, DFMA, and prototype-to-production ownership.

Emphasize:
- Siemens NX (Expressions, MBD), SolidWorks, Rhino
- GD&T (ASME Y14.5), tolerance stacks, DFMA
- Rapid prototyping, 3D printing, vendor coordination, AR hardware work

## 5. Rust, Cloud Robotics & Developer Infrastructure

Focus: backend, devops, and tooling work using Rust, cloud, and CI/CD to support robotics/industrial systems.

Representative JDs:

- Senior Rust Developer (example) (`rustfoundation-senior-rust-developer.md`)

When to use:
- Roles titled "Rust Engineer", "Backend Engineer (Rust)", "Robotics Cloud Infrastructure Engineer".
- Jobs that emphasize backend services, data pipelines, and dev tooling rather than direct robot control.

Emphasize:
- Rust (Tokio, Axum), Linux, Docker/Kubernetes
- CI/CD (GitHub Actions), AWS IoT / cloud infra
- Data collection, validation pipelines, and developer enablement for robotics teams
